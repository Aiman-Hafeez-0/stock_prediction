RETURN-PREDICT / RESEARCH NOTE
Quantitative Research — Model Comparison

Predicting next-day returns is mostly noise — the question is which model admits it honestly.

SVR, Gradient Boosting, and an LSTM, evaluated with walk-forward validation — retrained on an expanding window and tested only on the days immediately following each retrain — then compared against naive baselines and two ensembling strategies.

Method

Validation

Walk-forward, expanding window

Rather than one fixed train/test split, the model retrains on each new fold and is scored only on the days immediately after — closer to how it would actually be deployed, and it prevents one lucky split from flattering a model.

Baselines

Two dumb benchmarks

Predict-zero and predict-yesterday's-return are scored on the exact same out-of-sample points as the ML models. If a model can't beat these, its added complexity isn't earning its keep.

Ensembling

Average + learned blend

A simple average of the three models, and a non-negative linear blend (stacking) fit on the first part of the out-of-sample period and tested only on the remainder — so the blend never sees its own test data while fitting.

Backtest

Sign-of-prediction, cost-aware

Each day's position is the sign of the predicted return (long / short / flat), with a 5bps cost charged on every position change. This is intentionally simple — no position sizing, no slippage model, no borrow cost — a lower bound on real-world friction, not an upper one.

Reading a negative R² honestly: next-day equity returns are dominated by information that isn't in price/volume history at all — news, macro surprises, order flow. When a naive predict-zero baseline beats every ML model on RMSE, that's not a failed experiment — it's the finding: for this feature set and horizon, model complexity isn't buying real predictive power, and a rigorous comparison is what surfaces that rather than hiding it.
